使用tea给自己的gitea提交PR

本文最后更新于：2026年5月7日 晚上

前言

在自建 Gitea 仓库中进行代码协作时，通过命令行工具提交 Pull Request 可以大大提高工作效率。tea 是 Gitea 官方推荐的命令行客户端，支持丰富的操作功能。本文详细介绍如何使用 tea 工具向自己的 Gitea 仓库提交 PR，无论是配置登录信息还是执行具体的提交流程，都能帮助你快速上手。

创建令牌

1.打开内网 Gitea → 头像 → 设置 → 应用

2.新建令牌，勾选这 3 个权限（其他全不勾）：

  • issue → 可读
  • repository → 读写
  • user → 可读

第一部分：TEA 初始化配置（仅第一次配置）

1. 删除旧的登录配置（清空残留信息）

2. 绑定内网 Gitea 账号（替换成你的令牌）

✅ 出现 Login as chziyue successful 代表配置完成

第二部分：提交 PR

前提：代码已写完，执行过 git add . + git commit -m "备注"

1. 创建新分支（禁止用 main 分支提 PR）

2. 推送分支到远程仓库

3. 创建 PR

第三部分：参数说明

  • --head test：当前的 PR 分支
  • --base main：要合并的目标分支（主分支）
  • --title "PR测试"：PR 的标题（可自定义）

成功标志

终端直接输出 PR 编号 + 链接，代表提交完成！
